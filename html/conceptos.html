Estructura

Tablas, registros y campos

Toda base de datos se organiza en tres elementos principales. Entenderlos es el primer paso para trabajar con cualquier sistema de datos.

Tabla Conjunto de datos organizado en filas y columnas. Cada tabla representa un tema, por ejemplo: Clientes, Productos o Pedidos.
Registro Cada fila de la tabla. Representa una entrada única de información, por ejemplo: los datos de un cliente específico.
Campo Cada columna de la tabla. Define un tipo de dato concreto, por ejemplo: nombre, apellido, correo o fecha de registro.

Datos

Tipos de datos en un campo

Cada campo debe tener asignado un tipo de dato. Esto indica qué clase de información puede almacenarse en esa columna y cómo se comporta al consultar o procesar datos.

Texto Nombres, descripciones, correos electrónicos. Ej: VARCHAR, TEXT
Número Cantidades, precios, edades. Ej: INT, DECIMAL, FLOAT
Fecha / Hora Fechas de registro o vencimiento. Ej: DATE, DATETIME
Booleano Valores de verdadero o falso. Ej: BOOLEAN (TRUE / FALSE)

Elegir el tipo de dato correcto garantiza un almacenamiento adecuado, evita errores y permite que la base de datos optimice sus consultas.

Claves

Clave primaria y clave foránea

Las claves son campos especiales que permiten identificar registros y conectar tablas entre sí.

Clave primaria (Primary Key) Identifica de forma única cada registro de una tabla. No puede repetirse ni estar vacía. Ejemplo: el ID de un cliente.
Clave foránea (Foreign Key) Campo de una tabla que hace referencia a la clave primaria de otra. Establece la relación entre tablas y mantiene la integridad de los datos.
Operaciones

Operaciones básicas: CRUD

CRUD es el acrónimo de las cuatro operaciones fundamentales que se realizan sobre los datos. Toda aplicación que gestiona información las utiliza.

C Create

Insertar nuevos registros. Comando: INSERT

R Read

Consultar y leer datos. Comando: SELECT

U Update

Modificar registros existentes. Comando: UPDATE

D Delete

Eliminar registros. Comando: DELETE

CRUD representa las cuatro acciones básicas que se realizan sobre datos almacenados: crear, leer, actualizar y eliminar.
