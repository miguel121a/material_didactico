Creación de la Base de Datos

Para el desarrollo del proyecto se creó una base de datos relacional denominada u941347256_proyectEstadia, cuyo objetivo es almacenar y organizar la información académica de los estudiantes de manera eficiente.

El diseño se realizó siguiendo el modelo relacional, el cual permite que la información se distribuya en diferentes tablas relacionadas entre sí mediante claves primarias y claves foráneas. Esto ayuda a evitar la duplicidad de datos, facilita las consultas y mejora la integridad de la información almacenada.

Antes de crear la base de datos se identificaron las entidades principales involucradas en el sistema. Posteriormente se definieron los atributos de cada entidad y las relaciones existentes entre ellas.

Entidades identificadas

  • Estudiantes

    Entidad central del sistema. Almacena datos personales como nombre, CURP, fecha de nacimiento, dirección y teléfono de cada alumno registrado.

  • Docentes

    Registra la información del personal docente: nombre, apellidos, clave de trabajador, correo electrónico y teléfono de contacto.

  • Materias

    Contiene las materias impartidas en la institución, vinculadas a su docente correspondiente mediante una clave foránea.

  • Grupos

    Define los grupos escolares disponibles, organizados por grado y letra identificadora, a los que se asignan los estudiantes.

Herramientas utilizadas

La creación de la base de datos se llevó a cabo utilizando MySQL a través de phpMyAdmin, donde se generaron las tablas necesarias, se configuraron los campos correspondientes y se establecieron las relaciones para garantizar una correcta organización de los datos.

Proceso de Subida a la Nube

Una vez terminada la base de datos en entorno local, se realizó el proceso de migración hacia Hostinger para que el proyecto pudiera funcionar en la nube. El proceso se dividió en dos fases principales.

Fase 1 — Exportación desde phpMyAdmin local

Primero se exportó la base de datos desde phpMyAdmin instalado de manera local con XAMPP. Se utilizó la opción de exportación en formato SQL, lo que generó un archivo con todas las instrucciones necesarias para recrear las tablas y los datos en cualquier otro servidor.

  • Paso 1

    Abrir phpMyAdmin desde el panel de control de XAMPP.

  • Paso 2

    Seleccionar la base de datos u941347256_proyectEstadia en el panel izquierdo.

  • Paso 3

    Ir a la pestaña Exportar en la barra de navegación superior.

  • Paso 4

    Elegir el método de exportación Rápido y el formato SQL.

  • Paso 5

    Hacer clic en Exportar y guardar el archivo .sql generado en el equipo.

Fase 2 — Importación en Hostinger

En Hostinger el proceso fue mucho más sencillo porque la plataforma tiene phpMyAdmin integrado. Solo se creó la base de datos desde el panel de control, se abrió phpMyAdmin, se seleccionó la base de datos y se importó el archivo SQL directamente.

  • Paso 1

    Ingresar al panel de control de Hostinger y crear una nueva base de datos MySQL.

  • Paso 2

    Acceder a phpMyAdmin desde el apartado de bases de datos en hPanel.

  • Paso 3

    Seleccionar la base de datos recién creada en el panel izquierdo.

  • Paso 4

    Ir a la pestaña Importar y seleccionar el archivo .sql exportado previamente.

  • Paso 5

    Hacer clic en Importar y esperar a que el proceso finalice correctamente.

Al finalizar, la importación se completó con éxito sin ningún error, quedando la base de datos disponible en la nube con todas sus tablas y datos correctamente configurados.
